# 데이터 설명 (Data Documentation)

본 디렉토리는 본 프로젝트에서 사용한 데이터의 **구조와 설계 방식**을 설명한다.  
원본 데이터는 용량 및 라이선스 이슈로 인해 GitHub에 포함하지 않으며,  
데이터 처리 흐름과 최종 결과 구조를 중심으로 정리하였다.

---

## 데이터 구성 개요

본 프로젝트의 데이터는 다음과 같은 단계로 구성된다.

1. **원본 데이터 (raw)**
   - 기상청 날씨 관측 데이터
   - 서울교통공사 지하철 혼잡도 및 승하차 인원 데이터
   - 공휴일 및 코로나 관련 외부 자료

2. **병합용 베이스 데이터 (비공개)**
   - 기상 데이터 기준 시간축을 고정하기 위한 베이스 테이블
   - 지하철 데이터를 매칭하기 위한 중간 단계 데이터

3. **전처리 완료 데이터 (processed)**
   - 모델 입력에 사용되는 최종 데이터
   - GitHub에는 구조 확인을 위한 샘플만 제공

---

## 원본 데이터 (raw)

### 기상 데이터
- 출처: 기상청
- 시간 단위 관측 데이터
- 사용한 주요 원본 변수:
  - 기온
  - 풍속
  - 습도
  - 강수량
  - 일사량

### 지하철 데이터
- 출처: 서울교통공사
- 역별·시간대별 승하차 인원
- 요일 구분별 혼잡도 비율(%)

### 외부 이벤트 데이터
- 공휴일 여부
- 코로나 관련 변수

해당 데이터들은 날짜 단위로 직접 조사하여 정합화하였다.

> 원본 데이터는 본 레포지토리에 포함하지 않는다.

---

## 기상 데이터 파생 변수 생성

본 프로젝트에서는 단순히 제공된 기상 변수만 사용하는 것이 아니라,  
**원본 관측 데이터를 기반으로 파생 기상 지표를 직접 계산**하여 사용하였다.

### 파생 변수
- **체감온도 (FeelsLikeTemp)**
- **습구온도 (TW)**

해당 변수들은 기온, 풍속, 습도 등의 원본 관측값을 기반으로  
기상 지표 산출 공식에 따라 계산하였다.

계산된 값은 엑셀 기반 수식으로 1차 산출 및 검증 후  
모델 입력 데이터에 반영하였다.

---

## 병합용 베이스 데이터 (비공개)

기상 데이터와 지하철 데이터를 안정적으로 병합하기 위해  
사전에 정의된 **시간 단위 기상 베이스 테이블**을 사용하였다.

- 학습 데이터용 베이스: **2021–2023년**
- 검증 데이터용 베이스: **2024년**

해당 베이스 데이터는:
- 기상 컬럼 구조를 고정하는 템플릿 역할을 하며
- 이후 지하철 혼잡도 및 승하차 데이터를 매칭하여
  최종 모델 입력 데이터가 생성된다.

이 베이스 데이터는 중간 산출물로서  
GitHub에는 포함하지 않는다.

## 혼잡도 재구성 방식

열차 편성 수 및 정확한 수용 인원 정보가 제공되지 않는 문제를 해결하기 위해,  
과거 승차 인원과 공식 혼잡도 비율을 활용하여  
**기준 수용 인원(reference capacity)**을 역산하였다.

기준 수용 인원 = 실제 승차 인원 ÷ (혼잡도 / 100)


이를 통해 시간대별 혼잡도를 일관된 기준으로 재구성하였으며,  
inner / outer 혼잡도로 분리하여 사용하였다.

## 전처리 완료 데이터 (processed)

전처리 완료 데이터는 다음 특징을 가진다.

- 단위: 역 × 시간
- 포함 정보:
  - 기상 변수 및 파생 지표
  - 시간/요일 정보
  - 외부 이벤트 변수
  - 1시간 전 / 24시간 전 혼잡도

GitHub에는 전체 데이터 대신  
**구조 확인 및 재현을 위한 샘플 CSV만 제공**한다.

---

## 참고 사항

- 원본 데이터 및 병합용 베이스 데이터는 공개하지 않는다.
- 데이터 흐름 및 전처리 로직은 `preprocessing/` 디렉토리의 코드로 재현 가능하다.
